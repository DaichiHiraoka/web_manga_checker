## 機能追加: 履歴の日付フィルタリング

### 目的
ユーザーがWebマンガ閲覧履歴を特定の日付範囲で絞り込めるようにすることで、履歴の視認性と管理性を向上させます。

### 変更点
- `frontend/src/App.jsx` を変更しました。
- 日付フィルタリングの状態 (`startDate`, `endDate`) を管理するためのReact `useState` フックを追加しました。
- 履歴データ (`history`) のフィルタリングロジックを更新し、`startDate` と `endDate` に基づいて履歴エントリを絞り込むようにしました。
    - `startDate` が設定されている場合、その日付以降の履歴のみを表示します。
    - `endDate` が設定されている場合、その日付の終わり（翌日の0時）までの履歴を表示します。
- 履歴表示部分に、日付選択用の入力フィールド（`<input type="date">` 要素）を追加しました。

### テストの方法
1.  **フロントエンド開発サーバーの起動:**
    `web_manga_checker` プロジェクトのルートディレクトリで、以下のコマンドを実行してフロントエンド開発サーバーを起動します。

    ```bash
    cd frontend
    npm run dev
    ```

2.  **ブラウザでのアクセス:**
    ブラウザで表示されたURL（通常は `http://localhost:5173` など）にアクセスします。

3.  **履歴ページの確認:**
    アプリケーションの履歴表示セクションに移動します。

4.  **日付フィルタリング機能の操作:**
    - 履歴リストの上に新しく追加された「Start Date:」と「End Date:」というラベルと日付入力フィールドが表示されていることを確認します。
    - 「Start Date:」フィールドに開始日を入力します。
    - 「End Date:」フィールドに終了日を入力します。
    - 必要に応じて、サイトフィルタリングと組み合わせて動作を確認します。

### テストの成功基準
- 日付入力フィールドが正しく表示されること。
- 「Start Date:」のみが設定された場合、その日付以降のすべての履歴が表示されること。
- 「End Date:」のみが設定された場合、その日付の終わりまでのすべての履歴が表示されること。
- 「Start Date:」と「End Date:」の両方が設定された場合、指定された日付範囲内の履歴のみが表示されること。
- 07/01 start と 07/01 end を設定した場合、07/01のデータのみが表示されること。
- 日付フィルタリングの切り替えがスムーズに行われ、表示される履歴が即座に更新されること。
- フィルタリングされた履歴がない場合、「No history found for the selected site.」というメッセージ（または類似のメッセージ）が表示されること。
