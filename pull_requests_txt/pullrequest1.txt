## 機能追加 Webサイトごとの履歴フィルタリング

### 目的
ユーザーがWebマンガ閲覧履歴を特定のWebサイトで絞り込めるようにすることで、履歴の視認性と管理性
を向上させます。

### 変更点
- `frontendsrcApp.jsx` を変更しました。
- フィルタリングの状態 (`selectedSite`) とユニークなサイトのリスト (`uniqueSites`)
を管理するためのReact `useState` フックを追加しました。
- 履歴データ (`history`) の取得後、`useEffect`
フック内で閲覧履歴からユニークなWebサイトのオリジン（例 `httpsexample.com`）を抽出し、
`uniqueSites` ステートに保存するようにしました。
- 履歴表示部分に、ユニークなサイトのリストをオプションとするドロップダウン（`select`
要素）を追加しました。
- ドロップダウンで選択されたサイトに基づいて履歴データをフィルタリングし、表示される履歴リストを
動的に更新するようにロジックを変更しました。

### テストの方法
1.  フロントエンド開発サーバーの起動
`web_manga_checker`
プロジェクトのルートディレクトリで、以下のコマンドを実行してフロントエンド開発サーバーを起動し
ます。

``` 
cd frontend 
npm run dev 
```


2.  ブラウザでのアクセス
ブラウザで表示されたURL（通常は `httplocalhost5173` など）にアクセスします。
3.  履歴ページの確認
アプリケーションの履歴表示セクションに移動します。
4.  フィルタリング機能の操作
- 履歴リストの上に新しく追加された「Filter by
Site」というラベルとドロップダウンが表示されていることを確認します。
-ドロップダウンをクリックし、履歴に存在するWebサイトのURL（オリジン）がリストされていることを確
認します。また、「All Sites」オプションも存在することを確認します。
- リストから特定のWebサイトを選択します。
### テストの成功基準
- ドロップダウンリストが正しく表示され、履歴に存在するすべてのユニークなWebサイトのオリジンがオプ
ションとして表示されること。
- 「All Sites」オプションを選択した場合、すべての閲覧履歴が表示されること。
- 特定のWebサイトを選択した場合、そのWebサイトのURLに一致する履歴エントリのみが表示され、他のサイ
トの履歴は非表示になること。
- フィルタリングの切り替えがスムーズに行われ、表示される履歴が即座に更新されること。
- フィルタリングされた履歴がない場合、「No history found for the selected
site.」というメッセージが表示されること。

4.  フィルタリング機能の操作
- 履歴リストの上に新しく追加された「Filter by
Site」というラベルとドロップダウンが表示されていることを確認します。
