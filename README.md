# web_manga_checker

## 概要

`web_manga_checker`は、閲覧したWebマンガの履歴を自動で記録し、ブラウザ上で管理・確認できる常駐アプリケーションです。

## 主な機能

*   **閲覧履歴の自動記録:** ブラウザ拡張機能が、指定されたWebマンガサイトの閲覧履歴（URL、タイトル）を自動で記録します。
*   **履歴のWeb UI:** 記録した履歴を一覧表示するWebインターフェースを提供します。
*   **監視対象サイトの管理:** どのサイトを記録対象とするか、Web UI上から簡単に追加・削除できます。

## 技術スタック

本プロジェクトは、以下の技術を使用して構築されています。

*   **バックエンド:**
    *   [Rust](https://www.rust-lang.org/)
    *   [Axum](https://github.com/tokio-rs/axum): Webフレームワーク
    *   [Rusqlite](https://github.com/rusqlite/rusqlite): SQLiteデータベース操作
*   **フロントエンド:**
    *   [React](https://reactjs.org/)
    *   [Vite](https://vitejs.dev/): ビルドツール
*   **ブラウザ拡張機能:**
    *   JavaScript (Manifest V3)

## アーキテクチャ

本アプリケーションは、3つの主要コンポーネントで構成されています。

1.  **バックエンド (Rust):**
    APIサーバーとして機能し、データベースの管理、履歴の保存、フロントエンドへのデータ提供など、すべてのコアロジックを担当します。
2.  **ブラウザ拡張機能 (JavaScript):**
    ブラウザの「センサー」として動作します。ユーザーが閲覧しているページの情報を取得し、それが監視対象サイトであればバックエンドに通知して履歴を記録させます。
3.  **フロントエンド (React):**
    バックエンドから取得した履歴データを表示し、ユーザーが監視サイトを管理するためのWebインターフェースを提供します。

## セットアップ方法

### 前提条件

*   [Rust](https://www.rust-lang.org/tools/install)
*   [Node.js](https://nodejs.org/en/)

### 1. バックエンドの起動

```bash
# プロジェクトルートディレクトリに移動
# cd web_manga_checker

# データベースの初期化とサーバーの起動
cargo run
```
サーバーは `http://127.0.0.1:3000` で起動します。

### 2. フロントエンドの起動

```bash
# フロントエンドディレクトリに移動
cd frontend

# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```
フロントエンドは `http://localhost:5173` など、ターミナルに表示されたアドレスで起動します。

### 3. ブラウザ拡張機能の導入 (Google Chrome)

1.  Chromeで `chrome://extensions/` を開きます。
2.  右上の「デベロッパーモード」をオンにします。
3.  「パッケージ化されていない拡張機能を読み込む」ボタンをクリックします。
4.  ファイル選択ダイアログで、このプロジェクトの `extension` ディレクトリを選択します。
5.  「Web Manga Checker」が拡張機能として追加され、動作を開始します。
